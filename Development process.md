# 开发流程规范

## 核心原则

1. 代码写完 ≠ 任务完成
2. 推送前必须确认用户
3. 功能必须验证可用后才能算完成

---

## 开发流程

```
需求 → 编码 → 验证 → 部署 → 自动化 → 用户验收
          ↑
      这一步不能跳过
```

---

## 验证清单

每次实现功能后，必须确认：

```
✅ 验证清单：
- [ ] 代码已测试通过
- [ ] 部署已完成
- [ ] 手动验证功能可用
- [ ] 自动化已设置
- [ ] 用户确认可以结束
```

---

## 推送规范

**必须确认后才能推送！**

推送前输出：
```
---
⚠️ 推送前确认：
- [ ] 代码已验证可用
- [ ] 用户已确认推送

请确认是否可以推送？
```

用户回复"可以"后才能执行 git push。

---

## 自动化规范

如果功能需要自动化运行（如cron定时任务）：

1. 在代码部署时就设置好自动化
2. 验证自动化确实在工作
3. 不要假设"以后再设置"

---

## 延迟判断

- ❌ 之前：代码写完 → "完成了"
- ✅ 之后：代码写完 → "已部署，请验证"

---

## 规则触发

- 任何功能开发
- 任何代码推送
- 任何自动化设置
